本项目主要目的是利用GFW漏洞，为PC和iOS设备提供一个最为便捷(但依赖个人服务器)的一个翻墙代理方案。以黑名单的方式记录需要代理的网站列表，最小化代理服务器的压力。

使用方法

PAC地址

http://106.186.122.102/25.pac 此PAC地址已经实效，仅用于展示案例。可以给我发邮件索取。

Windows

Internet选项 -> 连接 选项卡 -> 局域网设置(如果是电脑拨号上网, 这里点'设置') -> 使用自动配置脚本 -> 填入脚本地址: http://106.186.122.102/25.pac -> 确定

Mac OS X

系统设置 -> 网络 -> 高级 -> 代理 -> 自动代理配置 -> URL中填入脚本地址: http://106.186.122.102/25.pac -> 好

iOS (iPhone/iPad)

Wifi
  • 打开设置, 选择Wi-Fi
  • 选择当前使用的热点
  • 拖到最下面的代理设置，选择"自动"，填写PAC地址: http://106.186.122.102/25.pac
3G

3G网络比较麻烦，推荐使用shadowsocks客户端，内置shadowsocks代理（可能需要把App store 账号切换到美国区才能下载）

其它

上面介绍的是设置系统级代理。有些软件支持代理设置(典型的是 Firefox)，可以在应用内设置 PAC，而不影响其它软件。还有少数软件会忽略系统代理设置。
  • Android 上只能设置全局代理。不建议这么使用，流量大了可能会禁止代理访问国内网站。

    • 如果希望在本页面列出个人的代理服务器，请fork本项目并加入相关配置信息，再发pull request。直接给我发邮件亦可。

    其它代理方案推荐

    • VPN Gate: 公共VPN中继服务器, 支持多个VPN协议。
    • fqrouter: 目前主要在做Android 平台的系统代理以及无线中继(只有一部分Android设备支持中继模式)。
    • shadowsocks: 一个加密的 socks5 代理隧道，网站可以购买付费服务。shadowsocks.net上可以免费领取网友分享的账号。
